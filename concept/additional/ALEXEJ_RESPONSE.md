# Ответ на запрос Алексея — Taris: Платформа vs. Готовое решение

**Дата:** Март 2026  
**Документ:** Анализ требований и ответ потенциальному клиенту  
**Автор:** Sintaris

---

## 1. Суть возражения клиента

Алексей указал, что платформы для создания мультиагентных систем (LangGraph, AutoGen,
CrewAI, n8n и подобные) уже существуют и в принципе могут покрыть его потребности,
поэтому наше предложение кажется ему завышенным по цене.

**Это понимание неверно, и вот почему.**

---

## 2. Платформа ≠ Готовый продукт

Аналогия: **платформа для агентов — это двигатель и шасси автомобиля, но не
сам автомобиль.**

| Что предоставляет платформа (LangGraph / AutoGen / n8n) | Что нужно ещё создать для Алексея |
|---|---|
| Абстракции для описания агентов и цепочек вызовов | Конкретную логику каждого агента под его задачи |
| API для вызова LLM | Интеграцию с Telegram и голосовым вводом |
| Примитивы памяти (векторная БД, RAG) | Управление пользователем, хранение данных |
| Инструменты разработчика (Python-библиотеки) | Готовый пользовательский интерфейс |
| Возможность подключать инструменты | Конкретные агенты: календарь, почта, медицина |
| Абстрактную «память» | Длительное хранение истории и контекста в SQLite |
| — | Безопасность, аутентификацию, роли |
| — | Голосовой пайплайн (STT + TTS, офлайн на Raspberry Pi) |
| — | Деплой, поддержку, обновления |

**Итог:** Платформа — это инструмент для разработчика, создающего систему для
конечного пользователя. Алексей — конечный пользователь. Платформа сама по себе
не даст ему ни одной из перечисленных функций без существенной кастомной разработки.

## 4. Что требует дополнительной разработки для Алексея

### 4.1 Медицинский агент (3) — новый модуль

Это единственный функциональный агент из требований Алексея, которого **ещё нет**
в Taris. Требования от Алексея:

> «Анализ результатов обследований, посещений врачей с выдачей рекомендаций
> с учётом предыдущего контекста. Обучаться только по моим медданным.
> Специальным медицинским знаниям обучать не нужно — компетенции GPT достаточно.»

**Что нужно реализовать:**

| Компонент | Описание | Оценка трудозатрат |
|---|---|---|
| Загрузка мед. документов | Использовать существующий `bot_documents.py` — уже умеет PDF/DOCX/TXT | **0 ч** (уже есть) |
| Персональная база знаний | Пользователь загружает анализы, выписки — они индексируются в FTS5 | **0 ч** (уже есть) |
| Медицинский чат-агент | Специальный системный промпт для мед. контекста; инъекция релевантных фрагментов из документов пользователя | **4–8 ч** |
| Интерфейс агента (Telegram + Web) | Кнопка «💊 Мед. ассистент» → чат с RAG-контекстом из личных документов | **4–6 ч** |
| Временная шкала медданных | Хранение событий (визиты, анализы) с датами, вывод хронологии | **6–10 ч** |
| **Итого** | | **14–24 ч разработки** |

**Важно:** Алексей явно указал, что не нужно обучать модель специальным медицинским
знаниям — достаточно контекста из его собственных документов + GPT. Это **точно
описывает то, что уже делает `bot_documents.py`** — загрузка, индексирование и
использование документов как контекста для LLM. Нужно только создать специализированный
интерфейс.

### 4.2 Финансовый агент (будущее расширение)

Алексей упомянул возможность добавить финансовый блок в будущем. Это **архитектурно
уже поддерживается** — добавляется как новый модуль по существующему шаблону.

## 5. Обоснование стоимости

### Почему «взять платформу» не является бесплатным решением

Допустим, Алексей выбирает n8n (или LangGraph). Чтобы получить то, что уже есть
в Taris, ему нужно:

| Что нужно сделать | Трудозатраты (оценка) |
|---|---|
| Настройка Telegram-бота с меню и кнопками | 20–40 ч |
| Голосовой пайплайн (STT + TTS, офлайн) | 40–80 ч |
| Календарный агент с NL-вводом | 20–40 ч |
| Почтовый агент (IMAP, AI-дайджест) | 16–24 ч |
| Загрузка и использование документов (RAG) | 20–40 ч |
| Web UI (HTTPS, PWA, аутентификация) | 40–80 ч |
| Система памяти (SQLite, персист. контекст) | 10–20 ч |
| Безопасность, роли, деплой | 10–20 ч |
| Медицинский агент (специфика Алексея) | 14–24 ч |
| **Итого** | **190–368 ч** |

При ставке 2 500 руб/ч это **475 000 – 920 000 руб** только за разработку,
плюс время на поддержку, обновления и инфраструктуру.

**Наше предложение (250 000 – 350 000 руб)** — это стоимость **настройки
и расширения уже готовой системы**, в которой 90% функциональности Алексея
уже реализовано. Разница в цене объясняется именно этим.

## 6. Резюме для продаж

**Ключевые аргументы:**

1. **Платформы (AutoGen, n8n, LangGraph) — это инструменты разработчика,
   а не продукт для конечного пользователя.** Они не дают Алексею ни одной кнопки,
   ни одного голоса, ни одного письма из коробки.

2. **Taris — это уже работающий продукт**, который покрывает 8 из 9 явных
   требований Алексея. Нужно добавить только медицинский агент (14–24 ч).

3. **Стоимость разработки аналогичного решения с нуля на любой платформе**
   составит 475 000 – 920 000 руб. Мы предлагаем 250 000 – 350 000 руб
   за готовое решение плюс кастомизацию под его задачи.

4. **Медицинский агент** технически тривиален для Taris — инфраструктура
   (загрузка документов, RAG, контекст памяти) уже есть. Нужен только
   специализированный интерфейс и промпт.

5. **Модульная архитектура** гарантирует расширяемость: финансовый агент,
   интеграция с Google Calendar/Yandex и другие блоки добавляются инкрементально.
